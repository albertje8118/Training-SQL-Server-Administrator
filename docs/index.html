The NovaMart Challenge

NovaMart is a growing retail chain that suffered a catastrophic data loss incident. Management has approved a budget and hired you as their first DBA. Every lab in this course advances the story — from connecting to the server on Day 1 to importing the legacy product catalog on Day 2.

Lab Dependency Chain

Objects built in earlier labs are used in later labs 
Lab 01  →  Read-only (no objects created)
   │
Lab 02  →  Configures services & TCP/IP (no DB objects)
   │
Lab 03  →  Creates: NovaMart  (production DB)  +  NovaMart_Test (deleted at end)
   │
Lab 04  →  Creates: dbo.Department,  dbo.Employee  inside NovaMart
   │
Lab 05  →  Creates: login LabReader, user LabReader  →  tests access on NovaMart
   │
Lab 06  →  BACKUP DATABASE NovaMart  to  C:\SQLBackups\
   │
Lab 07  →  DBCC CHECKDB, index rebuild on NovaMart
   │
Lab 08  →  SQL Agent job: nightly backup of NovaMart
   │
Lab 09  →  Monitor NovaMart queries via DMVs
   │
Lab 11  →  Import CSV data INTO NovaMart

Lab Environment Checklist

Verify before starting each lab day
  • SQL Server service is Running (check in SQL Server Configuration Manager)
  • SQL Server Agent is Running
  • TCP/IP is Enabled (required from Lab 02 onward)
  • Connected in SSMS with a sysadmin login
  • NovaMart database exists (from Lab 03 onward)
  • dbo.Department and dbo.Employee tables exist (from Lab 04 onward)
