Module 08: SQL Server Agent & Automation

Lab 08 Jobs · Schedules · Alerts · Operators NovaMart Automation
Learning Objectives
  • Describe the components of SQL Server Agent and how they interact
  • Create a multi-step SQL Server Agent job using SSMS
  • Schedule a job using a recurring SQL Server Agent schedule
  • Monitor job execution history to confirm success or diagnose failure
  • Script a job using system stored procedures

8.1 SQL Server Agent Components

SQL Server Agent is a Windows service (SQLSERVERAGENT) that runs scheduled tasks on behalf of SQL Server. It must be running before any jobs can execute.

ComponentDescription
JobA named container of one or more ordered steps to be executed
Job StepA single executable unit within a job — T-SQL, PowerShell, SSIS, CmdExec, etc.
ScheduleDefines when (recurring, one-time, or on SQL Server startup) a job runs
AlertMonitors for SQL Server errors, performance conditions, or WMI events and triggers a response
OperatorA named contact (email, pager) that receives notifications on job success, failure, or alert trigger
⚠
Agent Must Be Running

In SSMS, check SQL Server Agent under the Management folder in Object Explorer. If it shows a red stop icon, right-click → Start. To ensure it starts automatically after a server reboot, set the Agent service Startup Type to Automatic in SQL Server Configuration Manager.

8.2 Creating a Job via SSMS

SSMS provides a graphical job editor. Use it to create and test jobs before scripting them to T-SQL for deployment.

StepAction in SSMS
1In Object Explorer, expand SQL Server Agent → Jobs
2Right-click Jobs → New Job…
3On the General page, enter a job name (e.g., NovaMart Nightly Backup)
4On the Steps page, click New — enter step name, select type T-SQL, choose database, enter command
5On the Schedules page, click New — choose frequency, start time, and end date
6Click OK to save — the job appears under the Jobs node
ℹ
Step On-Failure Action

For each job step, set the On failure action to Quit the job reporting failure unless you intentionally want execution to continue after an error. This ensures job history accurately reflects whether the task succeeded.

8.3 Schedules and Job History

A schedule can be attached to multiple jobs. After attaching a schedule, verify in Job Activity Monitor that the next run time is set correctly.

T-SQL — Query Job History
-- View history for all jobs
SELECT
  j.name   AS JobName,
  h.step_name,
  h.run_date,
  h.run_time,
  CASE h.run_status
    WHEN 0 THEN 'Failed'
    WHEN 1 THEN 'Succeeded'
    WHEN 3 THEN 'Cancelled'
    WHEN 5 THEN 'Unknown'
  END AS Status,
  h.message
FROM msdb.dbo.sysjobhistory h
JOIN msdb.dbo.sysjobs        j ON h.job_id = j.job_id
ORDER BY h.run_date DESC, h.run_time DESC;

-- View only failed steps
SELECT j.name, h.step_name, h.message
FROM msdb.dbo.sysjobhistory h
JOIN msdb.dbo.sysjobs j ON h.job_id = j.job_id
WHERE h.run_status = 0
ORDER BY h.run_date DESC;

8.4 Creating a Job via T-SQL

Scripting jobs to T-SQL is essential for version control, deployment across environments, and disaster recovery. SQL Server Agent jobs live in the msdb system database.

T-SQL — Create Job with Steps and Schedule
USE msdb;
GO

-- 1. Create the job
EXEC sp_add_job
  @job_name = N'NovaMart Nightly Backup';

-- 2. Add Step 1 — Full Backup
EXEC sp_add_jobstep
  @job_name   = N'NovaMart Nightly Backup',
  @step_name  = N'Full Backup',
  @subsystem  = N'TSQL',
  @database_name = N'master',
  @command    = N'BACKUP DATABASE NovaMart
TO DISK = ''C:\SQLBackup\NovaMart_Full.bak''
WITH FORMAT, INIT, COMPRESSION, STATS = 10;';

-- 3. Add Step 2 — Update Statistics
EXEC sp_add_jobstep
  @job_name   = N'NovaMart Nightly Backup',
  @step_name  = N'Update Statistics',
  @subsystem  = N'TSQL',
  @database_name = N'NovaMart',
  @command    = N'EXEC sp_updatestats;';

-- 4. Add Step 3 — Log Backup
EXEC sp_add_jobstep
  @job_name   = N'NovaMart Nightly Backup',
  @step_name  = N'Log Backup',
  @subsystem  = N'TSQL',
  @database_name = N'master',
  @command    = N'BACKUP LOG NovaMart
TO DISK = ''C:\SQLBackup\NovaMart_Log.bak''
WITH INIT, STATS = 10;';

-- 5. Create a daily schedule at 02:00
EXEC sp_add_schedule
  @schedule_name   = N'Nightly 02:00',
  @freq_type       = 4,       -- Daily
  @freq_interval   = 1,       -- Every 1 day
  @active_start_time = 020000; -- 02:00:00

-- 6. Attach schedule to job
EXEC sp_attach_schedule
  @job_name      = N'NovaMart Nightly Backup',
  @schedule_name = N'Nightly 02:00';

-- 7. Register job on this server
EXEC sp_add_jobserver
  @job_name   = N'NovaMart Nightly Backup',
  @server_name = @@SERVERNAME;
GO
ProcedurePurpose
sp_add_jobCreates the job container in msdb.dbo.sysjobs
sp_add_jobstepAdds an executable step to the job
sp_add_scheduleCreates a reusable schedule definition
sp_attach_scheduleLinks a schedule to a job
sp_add_jobserverRegisters the job to run on a specific instance
sp_start_jobManually starts a job without waiting for its schedule
Knowledge Check
  1. What Windows service must be running before SQL Server Agent jobs can execute?
  2. A job has three steps. Step 2 fails. What should happen if each step's On-Failure action is set to "Quit the job reporting failure"?
  3. Where in SSMS can you view the next scheduled run time for a job?
  4. Which system stored procedure manually starts a SQL Server Agent job?
  5. Job history is stored in which database?
  1. The SQL Server Agent Windows service (SQLSERVERAGENT or SQLAgent$InstanceName for named instances).
  2. The job stops at Step 2, Step 3 is not executed, and the job reports a failure status in job history.
  3. In Job Activity Monitor (right-click SQL Server Agent → View Job Activity) or by expanding SQL Server Agent → Jobs and viewing the job properties.
  4. EXEC msdb.dbo.sp_start_job @job_name = N'job name';
  5. The msdb system database — specifically in msdb.dbo.sysjobhistory.

Create and Schedule the NovaMart Nightly Backup Job
~45 min Requires: SQL Agent running · NovaMart DB · C:\SQLBackup SSMS · msdb
Mission Briefing — Lab 08

Automate NovaMart's Nightly Backup Routine

Budi wants the nightly backup to run without manual intervention. Create a three-step SQL Server Agent job that performs a full backup, updates statistics, and backs up the log. Schedule it for 02:00 daily, run it manually to confirm it works, then review the job history.

Part A — Verify SQL Server Agent Is Running

  1. In SSMS Object Explorer, expand the SQL Server Agent node. If the icon shows a red stop sign, right-click → Start.

    SQL Server Agent node shows a green running icon.
Part B — Create the Job via T-SQL

  1. Open a new query window and run the full job creation script from Section 8.4 above. Ensure SQL Server Agent is running first.

    Commands complete without error. The job appears under SQL Server Agent → Jobs in Object Explorer (refresh the node).
Part C — Run the Job Manually
  1. T-SQL
    USE msdb;
GO
EXEC sp_start_job @job_name = N'NovaMart Nightly Backup';
GO

  2. Wait 30 seconds, then check job status:

    T-SQL
    SELECT
  j.name, h.step_name,
  CASE h.run_status WHEN 1 THEN 'Succeeded' ELSE 'Failed' END AS Status,
  h.message
FROM msdb.dbo.sysjobhistory h
JOIN msdb.dbo.sysjobs j ON h.job_id = j.job_id
WHERE j.name = N'NovaMart Nightly Backup'
ORDER BY h.run_date DESC, h.run_time DESC;
    All three steps (Full Backup, Update Statistics, Log Backup) show Status = Succeeded.
Part D — Review Job History in SSMS

  1. In Object Explorer, expand SQL Server Agent → Jobs. Right-click NovaMart Nightly Backup → View History.

    Job History dialog shows green checkmarks for each step. Drill into each row to see the start time, duration, and output message.

Validation Checklist

  • SQL Server Agent service is running
  • Job "NovaMart Nightly Backup" exists under SQL Server Agent → Jobs
  • Job has three steps: Full Backup, Update Statistics, Log Backup
  • Job has schedule "Nightly 02:00" attached
  • Manual run via sp_start_job completed successfully
  • All steps show Succeeded in job history

Troubleshooting

ProblemLikely CauseFix
sp_add_job fails — job already existsJob was partially created in a previous attemptRun EXEC sp_delete_job @job_name = N'NovaMart Nightly Backup' then re-run the creation script
Step fails — cannot open backup deviceC:\SQLBackup does not exist or SQL Server service account lacks permissionCreate the directory and grant write permission to the SQL Server service account
sp_start_job returns "job was not found"Typo in job name, or connected to wrong instanceCheck job name in Object Explorer; verify the server connection
Log backup step fails — "not permitted under simple recovery"NovaMart is in SIMPLE recovery modelRun ALTER DATABASE NovaMart SET RECOVERY FULL
