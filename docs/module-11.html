11.1 SQL Server Import and Export Wizard

The Import and Export Wizard is the quickest way to move data between SQL Server, Excel, CSV files, and other OLE DB/ODBC sources without writing code. It generates an SSIS package under the hood.

StepAction
1 — LaunchSSMS → right-click database → Tasks → Import Data… or Export Data…
2 — Choose SourceSelect data source type (SQL Server Native Client, Flat File, Excel, etc.) and connection details
3 — Choose DestinationSelect destination (SQL Server Native Client for another DB, or Flat File for CSV)
4 — Specify Copy or QueryChoose "Copy a table" for full table copy, or "Write a query" for filtered/transformed data
5 — Map ColumnsReview column mappings — set data types, drop existing rows (truncate), or append
6 — Run or SaveRun immediately, or save as an SSIS .dtsx package for reuse
7 — ValidateCheck row count in the destination table after completion
ℹ
Excel Import Tip

When importing from Excel, the wizard reads column data types from the first 8 rows. If a column has mixed types (text and numbers), it may import as NULL. Use CSV instead of Excel for reliable bulk imports into SQL Server.

11.2 BULK INSERT

BULK INSERT is a T-SQL statement for high-performance import of flat files. It is faster than the Wizard for large files and is easily scriptable for SQL Agent jobs.

T-SQL — BULK INSERT from CSV
-- Import a CSV file into the ProductImport staging table
BULK INSERT dbo.ProductImport
FROM 'C:\Data\products.csv'
WITH (
  FIRSTROW        = 2,           -- skip header row
  FIELDTERMINATOR = ',',         -- comma-separated
  ROWTERMINATOR   = '\n',        -- Unix line endings (use '\r\n' for Windows)
  TABLOCK,                       -- table-level lock — faster for large files
  BATCHSIZE       = 5000,        -- commit every 5000 rows
  ERRORFILE       = 'C:\Data\products_errors.txt'  -- log rejected rows
);
GO

-- Verify import
SELECT COUNT(*) AS ImportedRows FROM dbo.ProductImport;
SELECT TOP 10 * FROM dbo.ProductImport;
OptionMeaning
FIRSTROW = 2Skip the first row (header). Use 1 if there is no header.
FIELDTERMINATORColumn delimiter — comma (,), tab (\t), pipe (|), etc.
ROWTERMINATORRow delimiter — \n for Unix, \r\n for Windows CSV files
TABLOCKTakes a table-level lock during import — significantly faster for large files
BATCHSIZECommits data in batches — limits transaction log growth on large imports
ERRORFILEPath to write rows that fail to import — useful for debugging
⚠
File Path Must Be Accessible to the SQL Server Service Account

BULK INSERT reads the file from the SQL Server process, not from your client machine. The path must be a local path on the SQL Server or a UNC path accessible to the SQL Server service account. Paths mapped as a drive letter on your PC will fail.

11.3 BCP Utility

BCP (Bulk Copy Program) is a command-line tool for high-speed data import and export. It runs on the client machine and transfers data over the network to/from SQL Server.

Command Prompt — BCP Export and Import
-- Export a full table to a CSV file (trusted Windows auth)
bcp NovaMart.dbo.Employee out "C:\Data\employee_export.csv" ^
    -S localhost -T -c -t "," -r "\n"

-- Export using a query
bcp "SELECT EmpID, FullName, Salary FROM NovaMart.dbo.Employee WHERE Salary > 5000000" ^
    queryout "C:\Data\high_earners.csv" ^
    -S localhost -T -c -t "," -r "\n"

-- Import from a CSV file
bcp NovaMart.dbo.ProductImport in "C:\Data\products.csv" ^
    -S localhost -T -c -t "," -r "\n" -F 2
SwitchMeaning
outExport from SQL Server to a file
inImport from a file into SQL Server
queryoutExport the result of a SELECT query to a file
-SServer name (and instance if named: -S server\instance)
-TUse trusted (Windows) authentication
-U / -PSQL Server login and password (alternative to -T)
-cCharacter mode — suitable for CSV/text files
-tField terminator
-rRow terminator
-F 2Start from row 2 (skip header during import)

11.4 ETL Concepts

ETL — Extract, Transform, Load — is the pattern used to move data between systems, especially from transactional databases to data warehouses or reporting systems.

StagePurposeExample
ExtractPull data from one or more source systemsQuery NovaMart Employee table; read a CSV file from HR
TransformClean, validate, deduplicate, and reshape the dataStandardise date formats; convert NULL salaries to 0; join with department lookup
LoadWrite the transformed data to the destinationINSERT into a warehouse fact table; BULK INSERT into a staging table
ℹ
SSIS — SQL Server Integration Services

SSIS is Microsoft's enterprise ETL platform, included with SQL Server Developer, Standard, and Enterprise editions. It provides a visual design surface for building complex data pipelines with built-in transformations, error handling, and logging. For small-scale data movement within a single SQL Server instance, BULK INSERT, BCP, and the Import/Export Wizard are sufficient.

11.5 MERGE Statement (Upsert)

MERGE performs INSERT, UPDATE, and DELETE in a single statement, matching source rows to target rows. It is the standard pattern for loading incremental data into a staging or production table.

T-SQL — MERGE (Upsert)
-- MERGE: update existing products, insert new ones
MERGE dbo.Product AS target
USING dbo.ProductImport AS source
  ON (target.ProductID = source.ProductID)

WHEN MATCHED THEN
  UPDATE SET
    target.ProductName = source.ProductName,
    target.Price       = source.Price,
    target.Stock       = source.Stock

WHEN NOT MATCHED BY TARGET THEN
  INSERT (ProductID, ProductName, Price, Stock)
  VALUES (source.ProductID, source.ProductName, source.Price, source.Stock)

WHEN NOT MATCHED BY SOURCE THEN
  DELETE;  -- remove products no longer in the source file
GO

-- Row count by operation
-- Use OUTPUT clause to capture what was changed:
MERGE dbo.Product AS target
USING dbo.ProductImport AS source ON target.ProductID = source.ProductID
WHEN MATCHED THEN UPDATE SET target.Price = source.Price
WHEN NOT MATCHED BY TARGET THEN INSERT (ProductID, Price) VALUES (source.ProductID, source.Price)
OUTPUT $action AS MergeAction, inserted.ProductID
INTO @MergeLog (MergeAction, ProductID);
⚠
MERGE Requires a Semicolon Terminator

The MERGE statement must end with a semicolon (;). Omitting it causes a syntax error. This is one of the few T-SQL statements where the terminator is required, not optional.

Knowledge Check
  1. What does FIRSTROW = 2 do in a BULK INSERT statement?
  2. Which BCP switch uses Windows (trusted) authentication?
  3. What are the three stages of an ETL process?
  4. In a MERGE statement, what does WHEN NOT MATCHED BY TARGET mean?
  5. Why is TABLOCK recommended in BULK INSERT for large files?
  1. It tells BULK INSERT to skip the first row of the file (the header row) and start importing from row 2.
  2. The -T switch uses trusted (Windows integrated) authentication.
  3. Extract (pull data from source), Transform (clean and reshape), Load (write to destination).
  4. The row exists in the source but does NOT yet exist in the target table — so a new row should be inserted (INSERT action).
  5. TABLOCK acquires a table-level lock for the duration of the import, which allows SQL Server to use minimal logging and batch commit efficiently, significantly reducing both import time and transaction log growth.

Mission Briefing — Lab 11

Load the New Product Catalogue into NovaMart

The purchasing team has sent a CSV file with new product data. Load it into NovaMart using two methods: the Import Wizard and BULK INSERT. Then export the Employee table to CSV using BCP, and test a MERGE upsert.

Part A — Prepare the Data File and Staging Table

  1. Create the data directory and CSV file:

    PowerShell
    New-Item -ItemType Directory -Force -Path 'C:\Data'

$csv = @"
ProductID,ProductName,Category,Price,Stock
1,Laptop Pro 15,Electronics,15000000,50
2,Wireless Mouse,Peripherals,350000,200
3,USB Hub 7-Port,Peripherals,175000,300
4,Monitor 24in,Electronics,3200000,80
5,Keyboard Mechanical,Peripherals,850000,150
"@
$csv | Set-Content -Path 'C:\Data\products.csv' -Encoding UTF8

  2. Create the staging table in NovaMart:

    T-SQL
    USE NovaMart;
GO
CREATE TABLE dbo.ProductImport (
  ProductID   INT           NOT NULL,
  ProductName NVARCHAR(100) NOT NULL,
  Category    NVARCHAR(50)  NULL,
  Price       DECIMAL(15,2) NOT NULL,
  Stock       INT           NOT NULL,
  CONSTRAINT PK_ProductImport PRIMARY KEY (ProductID)
);
GO
Part B — Import via BULK INSERT
  1. T-SQL
    BULK INSERT dbo.ProductImport
FROM 'C:\Data\products.csv'
WITH (
  FIRSTROW        = 2,
  FIELDTERMINATOR = ',',
  ROWTERMINATOR   = '\n',
  TABLOCK,
  ERRORFILE       = 'C:\Data\products_errors.txt'
);
GO

SELECT COUNT(*) AS ImportedRows FROM dbo.ProductImport;
SELECT * FROM dbo.ProductImport;
    ImportedRows = 5. All five products appear in the result set.
Part C — Export Employee Table via BCP

  1. Open a Command Prompt (not PowerShell) and run:

    Command Prompt
    bcp NovaMart.dbo.Employee out "C:\Data\employee_export.csv" -S localhost -T -c -t "," -r "\n"
    BCP reports: N rows copied. File C:\Data\employee_export.csv created. Open it in Notepad to verify columns and rows.
Part D — Test Import via Wizard (Optional)

  1. In SSMS, right-click the NovaMart database → Tasks → Import Data…. Select Flat File Source, browse to C:\Data\products.csv, configure the destination as SQL Server Native Client → NovaMart → new table dbo.ProductWizard, and complete the wizard.

    Wizard reports all rows transferred. SELECT * FROM dbo.ProductWizard returns 5 rows.
Part E — Bonus: MERGE Upsert

  1. Create a Product target table and upsert from ProductImport:

    T-SQL
    -- Create target table
SELECT * INTO dbo.Product FROM dbo.ProductImport WHERE 1 = 0;
GO

-- Insert initial rows (Products 1, 2, 3 with old prices)
INSERT INTO dbo.Product VALUES
  (1, 'Laptop Pro 15', 'Electronics', 14000000, 40),
  (2, 'Wireless Mouse', 'Peripherals',  300000, 180),
  (3, 'USB Hub 7-Port', 'Peripherals',  150000, 250);
GO

-- Merge: update existing, insert new
MERGE dbo.Product AS target
USING dbo.ProductImport AS source
  ON (target.ProductID = source.ProductID)
WHEN MATCHED THEN
  UPDATE SET
    target.ProductName = source.ProductName,
    target.Price       = source.Price,
    target.Stock       = source.Stock
WHEN NOT MATCHED BY TARGET THEN
  INSERT (ProductID, ProductName, Category, Price, Stock)
  VALUES (source.ProductID, source.ProductName, source.Category,
          source.Price, source.Stock);
GO

-- Verify: Products 1-3 should have updated prices, 4-5 should be new
SELECT * FROM dbo.Product ORDER BY ProductID;
    Product 1 price is 15000000 (updated). Products 4 and 5 exist (inserted). All 5 rows present.

Validation Checklist

  • C:\Data\products.csv created with 5 product rows
  • dbo.ProductImport table created in NovaMart
  • BULK INSERT imported 5 rows successfully
  • BCP exported Employee table to C:\Data\employee_export.csv
  • (Bonus) MERGE updated existing products and inserted new ones
