Training Scenario —
NovaMart Digital Transformation

Role: New DBA Company: NovaMart 2-Day Course

The Company

NovaMart is a growing retail chain with 8 stores across the region, more than 200 employees, and a product catalog of over 5,000 items. The company sells consumer electronics, home appliances, and daily necessities.

Despite its growth, NovaMart's IT infrastructure has not kept pace. Critical business data is stored in a mix of Excel spreadsheets and an aging Microsoft Access database that lives on a single shared laptop.

The Problem

⚠
Critical Data Loss Incident

Last year, a hard drive failure destroyed 3 months of sales records and the entire product catalog. The company spent two weeks manually reconstructing data from paper receipts and supplier invoices.

Since then, the management team has approved a budget for proper database infrastructure. They need:

  • A reliable SQL Server instance with a backup strategy
  • Structured, consistent data with enforced business rules
  • Controlled access — not everyone should be able to edit everything
  • Automated maintenance so the database stays healthy without manual effort
  • A migration path to move existing Excel data into the new system

Your Role

You have just been hired as NovaMart's first dedicated Database Administrator. You report to Budi (IT Manager), who has given you a clear mandate: build, configure, secure, and maintain the SQL Server environment — and do it this week.

The Project Plan

Day 1 — Build the Foundation

ModuleYour Task
M01First day at the office. Budi walks you through the server room. Connect to the SQL Server instance and get familiar with the environment.
M02The production server needs proper configuration. Enable remote access so Sari and Dian can connect from their workstations.
M03Create the NovaMart production database with correct file sizing and recovery model.
M04Build the first core tables: Department and Employee. Verify that business rules are enforced.
M05Set up user access. Sari gets read-only. Agus gets read/write. Test that neither can exceed their permissions.

Day 2 — Protect, Automate, and Operate

ModuleYour Task
M06Design and implement a backup strategy for NovaMart. Simulate disaster recovery.
M07After a week of data entry, queries are slowing down. Rebuild indexes, update statistics, run DBCC CHECKDB.
M08Budi wants zero manual intervention for backups. Create a SQL Server Agent job that runs nightly at 11:00 PM.
M09Dian's monthly report takes over 5 minutes. Use Activity Monitor and DMVs to find what's blocking performance.
M10Monday morning. Three support tickets arrive at once: a login error, a slow server, and a remote connection problem.
M11Migration day. Import NovaMart's legacy product catalog from CSV. Export the Employee table to Excel for Sari's audit.

The Database We Are Building

By the end of Day 1, the NovaMart database will contain:

NovaMart database schema — Day 1
NovaMart (database) ├── dbo.Department — HR reference data (DeptID, DeptName, Location) └── dbo.Employee — Employee records (EmpID, Name, DeptID, Salary, HireDate)

Day 2 labs will work with this same database — adding backup jobs, running maintenance, monitoring queries, and importing additional data.

ℹ
Note on the Training Environment

In a real organization NovaMart would live on a production server protected by change control. In this training environment, you are the sysadmin and can work freely. Think of it as a pre-production environment.

Lab Progression

Each lab advances the NovaMart story. Objects created in earlier labs are used in later labs. Do not skip labs. If you fall behind, ask your instructor for the restore script so you can catch up.
