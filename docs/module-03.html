Module 03: Database Creation & Management

Lab 03 GUI & T-SQL Creates: NovaMart DB
Learning Objectives
  • Describe the physical structure of a SQL Server database (data files, log files)
  • Explain autogrowth settings and why default settings can cause problems
  • Create a database using both the SSMS GUI wizard and T-SQL
  • Explain what a filegroup is and when to use secondary filegroups
  • Describe SQL Server recovery models and their backup implications
  • Modify database properties after creation

3.1 Database Physical Structure

Every SQL Server database has at least one data file and one log file.

File RoleExtensionContains
Primary Data File.mdfData pages, system catalog, pointers to secondary files
Secondary Data File.ndfAdditional data pages — used when you want data on multiple disks
Transaction Log File.ldfAll transactions before they are committed — enables recovery

Autogrowth Settings

When a file runs out of space, SQL Server automatically expands it according to the autogrowth setting. Poor autogrowth causes fragmentation and performance spikes during busy periods.

⚠
Avoid Default Autogrowth

The default of 1 MB (data) and 10% (log) is too small for production. Use fixed MB increments — e.g., 256 MB for data files, 64 MB for log files. Always pre-size databases to avoid autogrowth during business hours.

3.2 Creating a Database with SSMS GUI

  1. In Object Explorer, right-click Databases → New Database…
  2. Enter the Database name.
  3. On the General page, configure the data and log file rows:
    • Change Initial Size to an appropriate starting value (e.g., 64 MB data, 16 MB log)
    • Click the Autogrowth ellipsis (...) → set to In Megabytes: 64
    • Set Maximum File Size to a reasonable cap (or leave Unlimited)
    • Change Path to your data/log directories if applicable
  4. Optionally, click the Options page to set recovery model, compatibility level, etc.
  5. Click OK.

3.3 Creating a Database with T-SQL

T-SQL — Minimal syntax
CREATE DATABASE NovaMart;
GO

SQL Server creates the data and log files in the default data directory with autogrowth defaults. For a controlled, production-ready database use the full syntax:

T-SQL — Explicit file specification
CREATE DATABASE NovaMart
ON PRIMARY (
    NAME        = NovaMart_Data,
    FILENAME    = 'C:\SQLData\NovaMart_Data.mdf',
    SIZE        = 64MB,
    MAXSIZE     = 10240MB,
    FILEGROWTH  = 64MB
)
LOG ON (
    NAME        = NovaMart_Log,
    FILENAME    = 'C:\SQLLogs\NovaMart_Log.ldf',
    SIZE        = 16MB,
    MAXSIZE     = 4096MB,
    FILEGROWTH  = 64MB
);
GO

3.4 Filegroups

Filegroups are logical containers that group one or more data files. Every database has at least the PRIMARY filegroup. Secondary filegroups are used to:

  • Spread data across multiple physical disks for I/O performance
  • Place large archive tables on slower (cheaper) storage
  • Partition large tables by filegroup (Enterprise feature)
T-SQL — Add a secondary filegroup
-- Add a filegroup
ALTER DATABASE NovaMart ADD FILEGROUP NovaMart_FG2;
GO

-- Add a data file to the new filegroup
ALTER DATABASE NovaMart
ADD FILE (
    NAME        = NovaMart_Data2,
    FILENAME    = 'C:\SQLData\NovaMart_Data2.ndf',
    SIZE        = 64MB,
    FILEGROWTH  = 64MB
) TO FILEGROUP NovaMart_FG2;
GO
ℹ
Training Note

The NovaMart lab uses the PRIMARY filegroup. You do not need secondary filegroups for today's work. This section is presented for awareness — you will encounter it when tables get large.

3.5 Recovery Models

The recovery model controls how the transaction log behaves and which backup strategies are available.

ModelLog BehaviourPoint-in-Time Recovery?Use When
SimpleLog is truncated after each checkpoint. Log file stays small.NoDev, test, or databases where losing a few minutes of data is acceptable
FullLog grows until a log backup is taken. All transactions retained.YesProduction databases that require point-in-time or log-shipping recovery
Bulk-LoggedMinimal logging for bulk operations. Log truncated after log backup.PartialBulk load windows (SSIS, BCP, bulk insert) — switch back to Full afterward
T-SQL — Change recovery model
ALTER DATABASE NovaMart SET RECOVERY FULL;
GO

-- Verify
SELECT name, recovery_model_desc
FROM   sys.databases
WHERE  name = 'NovaMart';

3.6 Database Properties

After creation, you can review and modify all settings in SSMS: right-click the database → Properties. Key pages:

PageKey Settings
GeneralStatus, owner, creation date, size, recovery model
FilesData and log file names, paths, sizes, autogrowth
FilegroupsList of filegroups; set default filegroup
OptionsRecovery model, compatibility level, AUTO_CLOSE, AUTO_SHRINK (avoid enabling AUTO_SHRINK in production)
PermissionsView and grant database-level permissions

3.7 Renaming and Dropping Databases

T-SQL
-- Rename (users must be disconnected first)
ALTER DATABASE NovaMart_Test MODIFY NAME = NovaMart_Test_Renamed;
GO

-- Drop (irreversible — ensure a backup exists first)
USE master;
GO
ALTER DATABASE NovaMart_Test_Renamed SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
GO
DROP DATABASE NovaMart_Test_Renamed;
GO
⚠
DROP DATABASE is Irreversible

Always verify you are dropping the correct database. In a production environment, ensure you have a backup before dropping any database.

3.8 Useful Database Queries

T-SQL
-- List all databases with size and recovery model
SELECT   name,
         recovery_model_desc,
         state_desc,
         CAST(SUM(size) * 8.0 / 1024 AS DECIMAL(10,2)) AS TotalSizeMB
FROM     sys.databases d
JOIN     sys.master_files mf ON d.database_id = mf.database_id
GROUP BY name, recovery_model_desc, state_desc
ORDER BY name;

-- View file details for a specific database
USE NovaMart;
SELECT   name, type_desc, physical_name,
         size * 8 / 1024  AS SizeMB,
         growth            AS AutogrowthPages,
         is_percent_growth AS GrowthIsPercent
FROM     sys.database_files;

Mission Briefing — Lab 03

Creating the NovaMart Production Database

Budi has approved the database name and the storage folder. He wants a production-grade setup: correct recovery model, reasonable file sizes, and controlled autogrowth. He also wants you to practice the SSMS GUI wizard first, then use T-SQL — which is what you will use from now on.

Part A — Create NovaMart_Test via SSMS GUI (Practice)

  1. In SSMS Object Explorer, right-click Databases → New Database…

  2. Set Database name = NovaMart_Test. In the database files grid, configure:

    FileInitial Size (MB)Autogrowth
    NovaMart_Test (data)32By 32 MB, max 2048 MB
    NovaMart_Test_log (log)8By 16 MB, max 512 MB

  3. Click the Options page. Note the Recovery model — it inherits from model (usually Simple). Click OK.

  4. Verify the database appears under Databases in Object Explorer.

    NovaMart_Test visible in Object Explorer.
Part B — Create NovaMart via T-SQL (Production Database)

  1. First, create the data and log directories (run in SSMS using xp_cmdshell or verify the folders exist in Windows Explorer). If the default SQL data folder exists, use it — adjust the path below as needed.

  2. Open a new query window and run:

    T-SQL
    CREATE DATABASE NovaMart
ON PRIMARY (
    NAME        = NovaMart_Data,
    FILENAME    = 'C:\Program Files\Microsoft SQL Server\MSSQL16.MSSQLSERVER\MSSQL\DATA\NovaMart_Data.mdf',
    SIZE        = 64MB,
    MAXSIZE     = 10240MB,
    FILEGROWTH  = 64MB
)
LOG ON (
    NAME        = NovaMart_Log,
    FILENAME    = 'C:\Program Files\Microsoft SQL Server\MSSQL16.MSSQLSERVER\MSSQL\DATA\NovaMart_Log.ldf',
    SIZE        = 16MB,
    MAXSIZE     = 2048MB,
    FILEGROWTH  = 64MB
);
GO
    Messages tab shows: Command(s) completed successfully.

  3. Set the recovery model to Full:

    T-SQL
    ALTER DATABASE NovaMart SET RECOVERY FULL;
GO
Part C — Verify the Database
  1. T-SQL
    SELECT name, recovery_model_desc, state_desc
FROM   sys.databases
WHERE  name IN ('NovaMart', 'NovaMart_Test');

USE NovaMart;
SELECT name, type_desc, physical_name,
       size * 8 / 1024  AS SizeMB,
       growth            AS AutogrowthPages
FROM   sys.database_files;
    NovaMart shows recovery_model_desc = FULL. Both files listed with correct sizes.
Part D — Modify File Size (Autogrowth Practice)

  1. Expand the log file to 32 MB using T-SQL:

    T-SQL
    USE master;
ALTER DATABASE NovaMart
MODIFY FILE (
    NAME       = NovaMart_Log,
    SIZE       = 32MB
);
GO
    Command completed successfully. Re-run the file query from Part C to confirm new size.
Part E — Delete the Test Database

  1. Drop NovaMart_Test — we only need NovaMart:

    T-SQL
    USE master;
GO
ALTER DATABASE NovaMart_Test SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
GO
DROP DATABASE NovaMart_Test;
GO
    NovaMart_Test disappears from Object Explorer. Only NovaMart remains under user databases.

Validation Checklist

  • NovaMart database appears in Object Explorer under Databases
  • Recovery model is FULL
  • Data file autogrowth = 64 MB (not percentage)
  • Log file size = 32 MB after modification
  • NovaMart_Test has been dropped and is no longer visible
